Devops Defined

“Devops is a way of thinking and a way of working. It is a framework for sharing stories and developing empathy, enabling people and teams to practice their crafts in effective and lasting ways. It is part of the cultural weave that shapes how we work and why. Many people think about devops as specific tools like Chef or Docker, but tools alone are not devops. What makes tools 'devops' is the manner of their use, not fundamental characteristics of the tools themselves.”
—Jennifer Davis & Ryn Daniels, 2016

State

Examples of infrastructure state:

  • Which version of PostgreSQL is installed?
  • How many servers are running?
  • Which version(s) of Python and GDAL are installed?

Here are examples of the state we are talking about

This can be confusing, difficult to troubleshoot, and often are hidden

State Simplified: Immutability

State is simplified by the principle of IMMUTABILITY

Don't Change, Have a great summer, etc.

Frontend Immutability: Redux

Embracing Immutable Architecture

The frontend has dealt with and solved this issue

Redux simplified the state of web applications

Infrastructure Immutability

  • Servers & Infrastructure as cattle, not pets
  • Don't modify/mutate - destroy and create new
  • Make persistent state (PostGIS) explicit and transparent

With a Devops approach

We can get immutability on the backend

This reduces the barriers to system maintenance and operations

Abstraction and State Management

Tools simplify by creating useful abstractions

A FOSS4G tool we may be familiar with is GDAL

GDAL does more than just this

But as a data abstraction library

It let's us simply interact with many formats using a common interface

ALL formats are exposed as an abstract "LAYER"

Terraform

The first tool, lowest in the tech stack

Is terraform

Terraform abstracts infrastructure & cloud providers

Similar to how GDAL abstracts data formats as layers

Terraform abstracts computing resources

Terraform State Managment

Infrastructure's state stored centrally (AWS S3) in JSON

Additionally, it makes our infrastructure state

Transparent and predictable

Having your infrastructure itself stored as version-controlled code

Is a huge win for simplicity

The Third Way: Continual Learning

“When new learnings are discovered locally, there must also be some mechanism to enable the rest of the [community] to use and benefit from that knowledge. In other words, when teams or individuals have experiences that create expertise, our goal is to convert that tacit knowledge (i.e., knowledge that is difficult to transfer to another person by means of writing it down or verbalizing) into explicit, codified knowledge, which becomes someone else’s expertise through practice.”

Gene Kim, The Devops Handbook: How to Create World-Class Agility, Reliability, & Security in Technology Organizations (Portland, OR: IT Revolution Press, LLC, 2016).
